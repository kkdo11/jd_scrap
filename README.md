# Wanted Job Matcher

> **원티드 채용공고를 내 이력서 기준으로 자동 채점해주는 CLI 도구**
> 로컬 LLM(Ollama)으로 돌리니까 **API 비용 0원**, 이력서·공고 데이터도 외부로 안 나갑니다.

매일 원티드 들어가서 공고 하나하나 클릭해서 "이거 내가 지원해도 되나?" 고민하는 게 귀찮아서 만들었습니다.
이력서 한 번 붙여넣어 두고 실행하면, 최신 공고 N개를 가져와서 0~100점으로 점수 매기고, **HTML 리포트로 정렬해서 보여줍니다.**

---

## 어떤 도구인가요?

1. 원티드 공식 API에서 **백엔드 / AI·ML / DevOps / 데이터엔지니어** 직군 최신 공고를 긁어옵니다.
2. 각 공고의 주요업무·자격요건·우대사항을 상세 조회합니다.
3. 내 `resume.txt`와 공고 내용을 LLM에 같이 던져서 **적합도 점수 + 매칭 포인트 + 부족한 부분 + 한줄평**을 받습니다.
4. 점수 높은 순으로 정렬한 **HTML 리포트(`report.html`)** 를 만들어줍니다.
5. 콘솔에는 TOP 5만 미리보기로 띄워줍니다.

병역특례 전용 공고, 신입 지원 불가 공고는 자동으로 걸러집니다.

---

## 사전 요구사항

### 소프트웨어
- **Node.js 18+**
- **[Ollama](https://ollama.com)** (로컬 LLM 실행 엔진. 무료, Mac/Windows/Linux 다 됩니다)

### 하드웨어 (LLM 돌리는 거라 GPU가 있는 게 좋습니다)
- **GPU 8GB+ VRAM** 권장 (예: RTX 3060 12GB, RTX 4070, RTX 4080 등)
- CPU만으로도 돌아가긴 하는데 공고 1개당 30초~1분씩 걸려서 비추.
- Mac은 M1/M2/M3 Apple Silicon이면 통합 메모리로 돌아갑니다 (16GB+ 권장).

### 추천 모델
- `exaone3.5:7.8b` — LG에서 만든 **한국어 네이티브** 모델, 약 4.8GB, 8GB GPU에 여유롭게 들어감. **기본 추천.**
  - ⚠️ 라이선스 NC(비상업) → 개인 사용만 OK
- `qwen2.5:14b` — Alibaba 모델, 한국어 출력 자연스러움 최상위, 16GB GPU에 들어감

## 설치 (처음 한 번만)

### 1. 이 레포 클론
```bash
git clone <이 레포 주소>
cd wanted-job-matcher
```

### 2. Node 의존성 설치
```bash
npm install
```

### 3. Ollama 설치
- Mac: `brew install ollama` 또는 [공식 사이트](https://ollama.com/download)에서 다운로드
- Linux: `curl -fsSL https://ollama.com/install.sh | sh`
- Windows: [공식 사이트](https://ollama.com/download)에서 인스톨러

### 4. 모델 다운로드 (한 번만, 약 5GB)
```bash
ollama pull exaone3.5:7.8b
```

설치 끝!

## ⚡ 원클릭 실행 (Windows + WSL)

매번 터미널 열고 명령 치기 귀찮다면, **더블클릭 한 번**으로 끝낼 수 있습니다.

### 준비 (처음 한 번)
1. 이 리포가 WSL의 `~/scrap`에 있다고 가정합니다. 다른 경로면 `win/원티드매칭.bat` 안의 `cd ~/scrap`를 실제 경로로 바꾸세요.
2. `win/원티드매칭.bat` 파일을 **Windows 바탕화면에 복사**하거나, 바로가기를 만드세요.
   - WSL에서 바탕화면으로 복사 예시:
     ```bash
     cp ~/scrap/win/원티드매칭.bat /mnt/c/Users/<윈도우계정>/Desktop/
     ```

### 실행
- 바탕화면의 **원티드매칭.bat 더블클릭** → 끝나면 브라우저에 `report.html`이 자동으로 뜹니다.

### 자동으로 해주는 것 (안정성 점검 + 자동 복구)
더블클릭하면 `run.sh`가 순서대로 점검하고, 빠진 건 알아서 복구합니다:

| 단계 | 점검 | 문제 시 |
| --- | --- | --- |
| 1 | Node.js/npm | 없으면 설치 안내 후 중단 |
| 2 | node_modules | 없으면 `npm install` 자동 |
| 3 | resume.txt | 없으면 템플릿 생성 후 "채우세요" 안내 |
| 4 | Ollama 설치 | 없으면 설치 명령 안내 후 중단 |
| 5 | Ollama 서버 | 꺼져 있으면 자동 시작 |
| 6 | 모델 | 없으면 자동 다운로드(~5GB) |
| 7 | 공고 분석 | 실행 |
| 8 | report.html | Windows 기본 브라우저로 자동 열기 |

> 진행 로그는 `run.log`에 쌓입니다. 문제가 생기면 이 파일을 확인하세요.

### 터미널에서 쓰고 싶다면
`.bat` 없이 WSL 터미널에서 직접:
```bash
./run.sh        # 기본 50개
./run.sh 20     # 20개만 빠르게
```

## 사용법

### Step 1. 이력서 준비
프로젝트 루트에 **`resume.txt`** 파일을 만들고 본인 이력서 내용을 텍스트로 붙여넣으세요.

> 💡 **팁:** PDF 이력서가 있다면 그냥 PDF 열어서 **Ctrl+A → Ctrl+C → resume.txt에 붙여넣기** 하면 됩니다.
> 형식 깨져도 LLM이 알아서 읽으니까 너무 신경 안 써도 돼요. 학력·경력·기술스택·프로젝트 위주로.

```bash
# 예시
nano resume.txt   # 또는 VSCode 등으로 편집
```

### Step 2. Ollama 서버 켜기
```bash
ollama serve
```
이미 시스템 트레이/백그라운드에서 실행 중이면 생략 가능. 확인하려면:
```bash
ollama list   # 받아둔 모델 목록이 뜨면 OK
```

### Step 3. 실행
```bash
npm start
```

기본 설정으로 **최근 공고 50개**를 분석합니다. 시간은 GPU에 따라 다른데, RTX 4080 기준 공고 1개당 5~10초 → 총 5~10분 정도.

### Step 4. 결과 확인
실행이 끝나면:
- 콘솔에 **TOP 5 추천 공고** 가 점수와 함께 출력됩니다.
- 프로젝트 폴더에 **`report.html`** 이 생성됩니다 → 브라우저로 열어서 보세요.

리포트에는 공고별로:
- 점수 (0~100)
- 매칭되는 내 스킬/경험
- 부족한 부분
- 한줄평
- 원티드 원본 링크
가 다 정리돼 있습니다.

## 웹으로 쓰기 (GUI)

CLI 대신 브라우저에서 쓰고 싶으면:

```bash
npm run web
```

그리고 브라우저에서 http://localhost:3000 접속.
이력서를 붙여넣고 공고 수를 정한 뒤 "분석 시작"을 누르면, 공고가 한 건씩
채점되며 실시간으로 카드가 채워집니다. (Ollama 서버가 켜져 있어야 합니다.)

> GPU가 1개라 한 번에 한 분석만 돌아갑니다. 실행 중 다시 누르면 "이미 실행 중"
> 안내가 뜹니다. 포트를 바꾸려면 `PORT=4000 npm run web`.

## 옵션 / 커스터마이징

### CLI 인자로 공고 수 조정
```bash
npx tsx src/index.ts 60    # 60개 분석
npx tsx src/index.ts 20    # 20개만 (빠르게 테스트할 때)
```

### 환경변수
| 변수 | 기본값 | 설명 |
| --- | --- | --- |
| `OLLAMA_MODEL` | `exaone3.5:7.8b` | 사용할 모델. 다른 모델 쓰려면 먼저 `ollama pull`로 받아두기 |
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama 서버 주소. 다른 PC에서 돌릴 때 사용 |
| `SCORER_CONCURRENCY` | `1` | 동시 요청 수. 단일 GPU는 1 권장(올려도 안 빨라짐) |
| `USE_CACHE` | `true` | 상세조회 결과 캐싱(`.cache/job-details.json`). `false`로 끄기 가능 |

예시:
```bash
# 다른 모델로 실행
OLLAMA_MODEL=qwen2.5:14b npm start

# 캐시 무시하고 새로 받기
USE_CACHE=false npm start
```

### 직군 변경
`src/wanted.ts`의 `TAG_IDS` 배열 수정. 기본값은 백엔드/AI/DevOps/데이터엔지니어:
```ts
const TAG_IDS = [872, 839, 940, 655];
// 872 = 서버/백엔드, 839 = AI/머신러닝, 940 = DevOps/인프라, 655 = 데이터엔지니어
```
다른 직군 ID는 원티드 사이트에서 해당 직군 필터 클릭 후 URL의 `tag_type_ids` 값 확인하면 됩니다.

### 경력 필터
`src/wanted.ts`의 `EXPERIENCE_YEARS`:
- `'-1'` = 경력무관(전체)
- `'0'` = 신입 (기본값)
- `'3'`, `'5'` 등 = N년차

신입 구직이면 `'0'`이 적당하고, 경력자면 본인 연차로 바꾸세요.

## 모델 선택 가이드

GPU VRAM 기준으로:

| VRAM | 추천 모델 | 비고 |
| --- | --- | --- |
| 8GB | `exaone3.5:7.8b` | 한국어 좋음, 빠름, **기본 추천** |
| 12GB | `exaone3.5:7.8b` 또는 `qwen2.5:14b`(q4) | 14B가 점수 보정 좀 더 정밀 |
| 16GB+ | `qwen2.5:14b` | 한국어 출력 자연스러움 최고 |

⚠️ **피해야 할 모델:**
- 32B 이상 (16GB GPU 초과 → CPU 오프로드로 매우 느려짐)
- **추론 모델** (EXAONE Deep, *-r1, qwen3 thinking 등) → 사고과정이 JSON에 섞여서 파싱 실패함

## 자주 묻는 질문

**Q. 점수가 너무 짠/후하게 나와요.**
A. 로컬 모델은 상용 모델보다 점수 보정이 거칠 수 있습니다. **절대 점수보다는 상대 순위·필터링 용도**로 보세요. TOP 5~10만 클릭해서 확인하는 게 효율적.

**Q. "Ollama 서버에 연결할 수 없습니다" 에러가 떠요.**
A. `ollama serve` 가 실행 중인지 확인하세요. 다른 터미널에서 `ollama list`가 정상 동작하면 OK.

**Q. 공고가 0개 나와요.**
A. 원티드 API 응답 구조가 바뀌었을 가능성. `src/wanted.ts`의 응답 파싱 부분 확인 필요.

**Q. 이력서가 외부로 새진 않나요?**
A. **안 새요.** Ollama는 100% 로컬에서 추론합니다. 외부로 나가는 네트워크 호출은 원티드 공고 API 호출뿐.

**Q. 더 빠르게 못 돌리나요?**
A. 단일 GPU에서 Ollama는 요청을 직렬 처리해서 동시성 올려도 안 빨라집니다. 진짜 throughput이 필요하면 **vLLM**의 continuous batching으로 바꾸는 게 정석. (`scorer.ts`를 OpenAI 호환 클라이언트로 교체)

**Q. 캐시는 뭐가 캐싱되나요?**
A. 원티드 상세 조회 결과(`.cache/job-details.json`). LLM 점수는 캐싱 안 함(이력서 바꾸면 다시 평가해야 하니까). 이력서 튜닝하면서 반복 실행할 때 네트워크 요청 줄여줍니다.

## 동작 원리 (간단히)

```
[원티드 API] → 공고 리스트 → 상세 조회 → [.cache 저장]
                                ↓
                         resume.txt + 공고 → [Ollama 로컬 LLM] → JSON (점수/매칭/갭/요약)
                                                                       ↓
                                                              [report.html 생성]
```

- 공고 수집: `src/wanted.ts`
- LLM 스코어링: `src/scorer.ts` (Ollama JSON 스키마 강제로 출력 안정화)
- HTML 리포트: `src/reporter.ts`
- 엔트리포인트: `src/index.ts`

## 주의사항

- 원티드 **비공식 내부 API** 를 사용합니다. 응답 구조가 바뀌면 `src/wanted.ts` 손봐야 할 수 있습니다.
- **개인 사용 목적으로만** 써주세요. 요청 사이에 250ms 딜레이 들어가 있어서 서버 부담은 거의 없지만 매너 지키기.
- EXAONE 3.5 라이선스는 **비상업(NC)** 입니다. 상업적 용도면 다른 모델로 바꾸세요.
