## 1. 전체 비전

원티드를 매일 들어가는 행동 자체를 없애는 게 목표.
**"새로 올라온 공고 중 내 핏에 맞는 거 N건이 알아서 도착"** — 친구가 매일 켜둘 필요 없이 GitHub Actions가 백그라운드에서 돌아가고, 결과는 푸시 알림으로 받는다.

### 핵심 가치
- **자동화**: 매일 정해진 시간에 알아서 실행
- **신호 추출**: 어제 본 공고 빼고 진짜 새 것만 보여줌
- **부담 제로**: 친구 PC가 꺼져있어도 동작, 무료 인프라만 사용

## 3. 범위 (Scope)

이번 사이클에서 다룰 4개 개선 항목:

### C. 이력서 프로필 요약 캐싱
**문제:** 40개 공고 평가 시 이력서를 40번 통째로 LLM에 전송 → 토큰·시간 낭비.
**해결:** 이력서를 1회만 LLM으로 분석해 구조화된 프로필 JSON(학력/경력/스택/연차)으로 요약하고, 매 공고 평가는 그 요약본만 같이 보냄.
**기대 효과:** 호출당 입력 토큰 60~80% 절감, 전체 실행 시간 2~3배 단축.

### D. "어제 본 공고"와 diff
**문제:** 매일 돌리면 어제 본 공고 90%가 그대로 결과에 또 나옴. 신규 식별 어려움.
**해결:** `.cache/seen.json`에 이미 평가한 공고 ID 저장. 리포트에 🆕 신규 배지 표시. 알림은 신규 + 고득점만.
**기대 효과:** "어제 봤던 거 또 보네" 현상 제거, 알림 노이즈 최소화.

### E. 임베딩 기반 사전 필터 + 2단계 평가
**문제:** 모든 공고를 무거운 LLM에 통과시킴. 명백한 미스매치도 30초씩 소비.
**해결:**
1. 이력서 임베딩 vs 공고 임베딩 cosine similarity 계산 (저렴/빠름, 0.5초/건)
2. 하위 30~50% 자동 탈락
3. 상위만 LLM 정밀 평가
**기대 효과:** 전체 속도 2배, 노이즈 공고 자동 제거.

### K. 알림 & 자동 실행
**문제:** 수동 트리거 방식 — 매일 터미널 열어서 npm start 해야 가치 발생.
**해결:** GitHub Actions cron으로 매일 자동 실행 + 결과를 webhook/메시지로 푸시.
**기대 효과:** "도구 사용" 자체가 무의식이 됨.

## 4. 결정 사항

### Q1: 어디서 돌릴 것인가? → ✅ **B (GitHub Actions + API fallback)**

선택 이유:
- 본인 PC가 꺼져있어도 동작
- 친구도 자기 레포에 fork하면 끝, GPU 불필요
- "매일 자동"의 핵심 가치를 손실 없이 달성

후속 영향:
- **API fallback 필수**가 됨 (GitHub Actions에는 GPU 없으므로 Ollama 사용 불가)
- 무료 티어 API provider 선택 필요 (Groq / OpenRouter / Anthropic credit / Gemini free 등)
- 로컬 Ollama 모드는 옵션으로 유지 (본인 PC에서 디버깅·튜닝 시 사용)

### Q2: 결과를 어떻게 받아볼 것인가? → ⏸️ **미결정 (답변 대기)**

후보:
- A. 디스코드 webhook + GitHub Pages **(추천)**
- B. 슬랙 webhook + GitHub Pages
- C. 이메일 (Resend) + GitHub Pages
- D. 텔레그램 봇
- E. GitHub Pages + RSS만 (알림 없음)

## 5. 남은 결정 사항 (다음 브레인스토밍 재개 시)

- **Q2: 알림 채널 + 리포트 호스팅 방식** (위)
- **Q3: API provider 선택** (Groq / OpenRouter / Anthropic / Gemini free 중 어느 것?)
  - 무료 한도, 한국어 품질, 모델 선택지, 지연 시간 비교 필요
- **Q4: 임베딩 모델은 어디서 돌릴 것인가?**
  - 옵션 1: provider 임베딩 API 같이 사용
  - 옵션 2: 가벼운 로컬 임베딩 (예: `text-embedding-3-small` 호환 무료 / Hugging Face inference free / sentence-transformers)
  - 옵션 3: GitHub Actions 안에서 CPU 임베딩 (1회 1초 미만, transformers.js)
- **Q5: 이력서를 어디 둘 것인가?**
  - GitHub Secrets에 base64? Private gist? Private repo만 허용?
  - 이력서는 절대 public commit 금지 — 인프라적으로 강제 방법 필요
- **Q6: 알림 트리거 조건**
  - 신규 공고 중 N점 이상만? 신규 + 점수 변동도 알림?
  - 매일 무조건 보낼지, 신규 없으면 침묵할지
- **Q7: cron 스케줄**
  - 매일 몇 시? 한국 시간 기준 아침 8시? 평일만? 주말 포함?
- **Q8: 기존 로컬 Ollama 경로 어떻게 처리?**
  - 완전 제거 vs 옵션으로 유지 (디버깅·튜닝 시 유용)
  - `--provider local|api` 같은 플래그?

## 6. 잠정 아키텍처 (변경 가능)

```
GitHub Actions (cron: daily 08:00 KST)
    ↓
[1] 원티드 API → 공고 N개 수집
    ↓
[2] 이미 본 공고 제거 (seen.json with diff)
    ↓
[3] 이력서 프로필 요약 (캐시: resume_profile.json, 이력서 해시 변하면 재생성)
    ↓
[4] 임베딩 사전 필터 (cosine sim < threshold → 자동 탈락)
    ↓
[5] LLM 정밀 평가 (API provider, JSON 강제)
    ↓
[6] 리포트 HTML 생성 → gh-pages 브랜치 푸시
    ↓
[7] 신규 + 고득점 → 알림 webhook
    ↓
[8] seen.json 업데이트 → 커밋
```

### 코드 변경 예상 범위
- `src/scorer.ts`: Ollama 클라이언트 → provider abstraction (`src/providers/`)
- `src/embedding.ts`: 신규 (임베딩 + cosine sim)
- `src/resume.ts`: 신규 (이력서 → 프로필 요약, 캐싱)
- `src/seen.ts`: 신규 (diff 관리)
- `src/notify.ts`: 신규 (알림 채널 어댑터)
- `.github/workflows/daily.yml`: 신규 (cron 워크플로)
- `src/wanted.ts`: 변경 적음 (URL/필터를 config로 빼는 작업은 이번 사이클에 안 함, 1번 트랙 보류)
