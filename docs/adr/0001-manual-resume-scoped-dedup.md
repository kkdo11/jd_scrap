# 0001. 중복 제거: 자동 대신 수동 체크 + 이력서별 네임스페이스

날짜: 2026-06-22 · 상태: 채택

## 골랐다
공고 중복 제거를 **사용자가 결과에서 "확인함" 체크한 것만 숨기는 수동 방식**으로,
체크 목록은 **이력서 텍스트 해시별 네임스페이스**로 보관(복원 가능).

## 왜
- 자동(가져온 것 전부 숨김)은 사용자가 아직 검토/지원 안 한 공고도 영영 안 보이게 해 **유실 위험**. 통제권을 사용자에게.
- 이력서가 바뀌면 매칭 기준이 달라져 **재평가가 자연스럽다**. 네임스페이스로 두면 여러 이력서를 번갈아 써도 각자 확인 상태 보존.

## 포기
- 자동 전역 dedup(원래 조각 C 메모의 의도). 단순하지만 유실 위험으로 기각.
- 전역 단일 seen 셋. 이력서 전환 시 의미가 깨져 기각.

## 재검토 시점
- 조각 D(자연어 쿼리) 도입 시 dedup이 "쿼리별"이어야 하는지 재검토. 현재는 이력서 키만으로 충분.
- 조각 B(멀티유저 원격 접속) 도입 시 네임스페이스에 사용자 차원 추가 필요.
